# AI 주식 포트폴리오 최적화 시스템 — 발표 초안(빠른 PPT 생성용)

발표 시간 10–15분 기준, 12장 내외. 각 슬라이드는 PowerPoint Copilot에 그대로 붙여도 무방하도록 구성했습니다.

---

## 1. 표지 (Title)
- AI 주식 포트폴리오 최적화 시스템
- 양자 컴퓨팅(QAOA) + 실시간 데이터 + 모던 웹앱
- 발표자: <이름/소속> | 날짜: 2025-11-10

스피커 노트:
- 한 줄 가치 제안: 실시간 가격과 위험 관리를 바탕으로 종목과 비중 결정을 자동화합니다.
- 웹 UI는 React, 백엔드는 Spring Boot, 최적화는 Python(Qiskit)으로 구성되어 있습니다.

---

## 2. 문제 정의 & 목표
- 개인 투자자는 종목 선택과 비중 결정이 어렵고 반복적
- 실시간 가격·환율·리스크를 함께 고려하기 복잡
- 목표: 손쉽게 종목을 담고 위험도/제약을 설정하면 최적 비중을 자동 추천

스피커 노트:
- 의사결정 시간을 줄이고 재현 가능한 프로세스를 제공합니다.

---

## 3. 솔루션 한눈에 보기
- 실시간 주가(yfinance)와 환율 반영(KRW 변환)
- QAOA 기반 포트폴리오 최적화(교육/연구 목적)
- 모던 UI(React+Tailwind)와 다국어/챗봇 지원

스피커 노트:
- 실제 양자 HW가 아니라 시뮬레이터로 QAOA를 사용하며, 교육/연구 목적임을 명시합니다.

---

## 4. 시스템 아키텍처
- Frontend: React 18 + Vite + Tailwind
- Backend: Spring Boot 3 (Java 17)
- Python: Qiskit Optimization, yfinance, NumPy/Pandas
- 통신: Spring -> Python 스크립트 실행(Commons Exec)

스피커 노트:
- 정적 프런트는 빌드되어 Spring의 static으로 서빙되며, API는 /api/*로 노출됩니다.

---

## 5. 주요 기능
- 종목 검색·추가, 수량/위험도 설정, 제약조건 반영
- 포트폴리오 최적화 실행 및 결과 시각화
- 실시간/폴백(Mock) 데이터 전략, 다국어, AI 챗봇

스피커 노트:
- 실패 시 Mock 데이터로 폴백되어 데모 안정성을 확보합니다.

---

## 6. 최적화 알고리즘(QAOA)
- 목적함수: minimize risk - risk_factor * return
- 제약: Σ weights = 1, 최소/최대 비중 등
- Qiskit Optimization + 시뮬레이터 기반

스피커 노트:
- 조합 최적화 특성상 QAOA 접근을 소개하고, 교육/연구 맥락을 강조합니다.

---

## 7. 데이터: 실시간 주가 & 환율
- yfinance 우선, 실패 시 Mock 폴백
- 해외 종목은 KRW 자동 변환(예: USD→KRW)
- 거래 시간/지연 특성 안내

스피커 노트:
- 데이터 소스 표기(yfinance/mock)와 로그 확인 포인트를 간단히 언급합니다.

---

## 8. 데모 플로우(사용자 여정)
- 1) 종목 검색 및 선택
- 2) 보유 수량/위험도/제약 설정
- 3) 최적화 실행 → 추천 비중/금액
- 4) 핵심 지표: 예상 수익률/위험도/샤프비율

스피커 노트:
- 실제 화면 캡처(대시보드, 최적화 결과)를 삽입하면 전달력이 크게 향상됩니다.
